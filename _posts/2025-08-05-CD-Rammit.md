But there was actually a connection from `ChromeSetup.exe` that connect to external IP Address, the reason why this connection interesting, it connected to unfamiliar port.

While, Chrome could connect to their infrastructure while downloading, it's should be in common port (**443**).

So it worth to spent a bit time to look into the process, also the file itself.

Before perform OSINT on this, i want to locate where the file is and it's metadata.

**MemProcFS** is stronger than **Vol3** on this one, locate the file at logical structure level. Since i used Forensic Mode 1 when analyze this `.dmp`, i could dig into where it located at `M:\forensic\ntfs\1\Users\alex\Downloads\ChromeSetup.exe`

#### Q5: Hashes serve as unique identifiers for files, assisting in the detection of similar threats across different machines. What is the SHA1 hash of the malware executable?

This question was pretty tricky, you will have multi hash values if you try to get it other source (For me, it was from MemProcFS Process ID - Handle).

The right approach that helped me find the answer was use Vol3 to dump entire `ChromeSetup.exe` process which is `--pid 4268` and calculated `ChromeSetup.exe` **ImageSectionObject** SHA1 Value (or simply throw the file into OSINT such as VirusTotal, Hybrid Analysis, etc).

